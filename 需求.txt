# 项目：基于 RK3566 NPU 的人脸识别门禁系统

## 硬件平台
- 主控：i.MX6UL，无 NPU，单核 Cortex-A7
- 摄像头：USB 摄像头，V4L2 驱动，输出 YUYV 格式，分辨率 640x480
- 显示屏：7寸 RGB 触摸屏，1024x600，GT911 触摸（通过 /dev/input/eventX 读取）
- 执行器：GPIO 控制一个 LED（模拟门锁），GPIO 编号可配置（假设使用 /dev/gpiochip0 line 17）

## 功能需求（B模式：识别特定身份后开门）

### 1. 视频采集模块（v4l2_capture）
- 使用 V4L2 API，mmap 零拷贝模式，至少 3 个缓冲区。
- 输出格式：V4L2_PIX_FMT_YUYV，分辨率 640x480。
- 提供接口获取最新一帧（环形队列 + 互斥锁/信号量）。

### 2. 显示模块（display）
- 将 YUYV 转换为 RGB565 写入 /dev/fb0（可先用软件转换，后期优化）。
- 集成 LVGL 图形库，实现以下 UI：
  - 实时摄像头预览（窗口）
  - 识别结果显示：“欢迎 [用户名]” 或 “未识别”
  - 开门记录列表（显示最近 10 条，时间+用户名）
- 触摸屏事件通过 /dev/input/eventX 读取并喂给 LVGL。

### 3. 人脸检测模块（face_detect）—— 使用 YOLOv5s 部署在 NPU
- 模型：YOLOv5s（或轻量级人脸检测模型如 RetinaFace），已转换为 RKNN 格式。
- 输入：RGB 图像（640x480），输出：人脸边界框坐标。
- 调用 RKNN API（librknnrt.so）进行推理。
- 只保留人脸类别（或单独的人脸检测模型）。

### 4. 人脸识别模块（face_recog）—— 使用 MobileFaceNet 部署在 NPU
- 模型：MobileFaceNet 预训练模型，输出 128 维特征向量，已转换为 RKNN 格式。
- 输入：人脸对齐后缩放到 112x112 RGB。
- 特征向量归一化（L2 范数）。
- 比对：计算与本地特征库中向量的欧氏距离，小于阈值（如 1.0）则识别成功。

### 5. 特征库管理（feature_db）
- 存储结构：每个用户一个 ID（整数），可存储 1~3 个特征向量（取平均或单独比对）。
- 存储方式：二进制文件（如 user_features.bin）或 SQLite。
- 提供接口：注册（添加特征）、删除、查询（比对返回 ID 或 -1）。
- 注册流程：采集 5 帧人脸，检测对齐后提取特征，存储特征向量的平均值。

### 6. 门控模块（gpio_control）
- 使用 libgpiod 控制 GPIO 输出高电平 2 秒后复位（模拟开门）。
- 提供接口：open_door()。

### 7. 主控逻辑（main）
- 创建三个线程：采集线程、识别线程、UI线程（LVGL 处理）。
- 采集线程获取 YUYV 帧，转换为 RGB 格式，放入共享队列。
- 识别线程取帧：
  - 调用人脸检测模型得到人脸框。
  - 若检测到人脸，裁剪并缩放到 112x112，送入识别模型得到特征向量。
  - 与特征库比对，若匹配成功则调用 open_door()，并记录日志（保存到文件）。
  - 若不匹配但有人脸，显示“未识别”。
- UI 线程响应触摸事件，处理注册、删除、查看记录等。

## 非功能性要求
- 代码使用 C，风格统一，关键函数注释详细。
- 使用 makefile 构建系统，支持交叉编译（arm-gcc-linux）。
- 提供 README.md，说明：
  - 依赖库（librknnrt, libgpiod, libjpeg, lvgl, opencv 等）
  - 编译步骤
  - 运行命令
  - 硬件接线（GPIO 引脚、触摸屏 I2C 等）
- 模型文件（.rknn）假设已放在 models/ 目录下，代码中需要加载并检查是否存在。
- 项目需要能够直接编译并在 RK3566 上运行（提供占位代码，但需保证整体逻辑完整）。
